# Dataset Schema

> **Locked files.** Machine-readable specs: `packages/shared/src/shared/schema/dataset/preflight_safe_record.json` for Step 5 training input, `packages/shared/src/shared/schema/dataset/dataset_record.json` for training output under `data/dataset/`, and `packages/shared/src/shared/schema/eval/` for held-out eval records/results.

The old shared train/eval corpus schema is intentionally gone. Step 5 training input is `PreflightSafeRecord`: `example_id` is the source/code identity and the row carries the preflight-selected tolerance policy. When the dataset loop hydrates that row, `dataset_id` is derived from code plus shapes, dtypes, RNG seed, op category, and tolerance policy. Held-out eval input is `EvalCorpusRecord`, which embeds `EvalSpec`, difficulty tier, and acceptance metadata. These are separate contracts because training and eval now carry different invariants.

Each row in the generated training dataset represents one complete PyTorch-to-Triton translation job. It contains a `source` block with immutable inputs (source `example_id`, PyTorch code, `origin`, input shapes, dtypes, RNG seed, op category, and the preflight-selected tolerance policy); a list of up to three `attempts`, each recording the generated Triton code, deterministic `patch_from_previous`, `/compile` result, optional `/run` error, deterministic `failure_symptom`, compact correctness outcome, and judge-derived `judge_classification`, `judge_root_cause`, `judge_repair_action`, and `judge_fix_suggestion` for failed attempts; and a top-level `final_outcome`.

`final_outcome` is derived from verification results, not from the judge label. `compiled_correct` requires at least one attempt whose correctness check passed. `numeric_fail` means at least one attempt reached numeric comparison, but no attempt passed tolerance. `runtime_fail` means at least one attempt passed `/compile` but `/run` failed before correctness was produced. `compile_fail` means no attempt passed `/compile`.

Training dataset rows do **not** record benchmark timings, speedups, or `final_winning_attempt_n`; benchmarking is eval-only. Every training row is guaranteed to have passed a pre-flight eager-vs-Inductor sanity check before the dataset loop starts. All categorical fields use closed vocabularies enforced as Python Enums in `packages/shared/src/shared/enums.py`; adding a value requires a decision-log entry and team sign-off before implementation. For corpus provenance and the train/eval split, see `docs/corpus.md`.
