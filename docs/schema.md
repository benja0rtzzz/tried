# Dataset Schema

> **Locked file.** Do not make changes. Machine-readable spec: `packages/shared/src/shared/schema/dataset/dataset_record.json`.

Each row in the dataset represents one complete PyTorch-to-Triton translation job. It contains a `source` block with the immutable inputs (PyTorch code, `origin` identifying where the code was extracted or curated from, input shapes, dtypes, RNG seed, and op category); a list of up to five `attempts`, each recording the generated Triton code, compile result, correctness stats against both PyTorch eager and Inductor, benchmark speedups, judge classification, and the judge's fix suggestion for the next retry; and a top-level `final_outcome` and `final_winning_attempt_n` that identify the terminal state and which attempt determined it. Every row is guaranteed to have passed a pre-flight eager-vs-Inductor sanity check run before the agent loop starts — examples that fail that check are written to `skipped.jsonl` and never enter the dataset. All categorical fields use closed vocabularies enforced as Python Enums in `packages/shared/src/shared/enums.py`; adding a value requires a decision-log entry and team sign-off before implementation. For corpus provenance, including curated training rows and the train/eval split, see `docs/corpus.md`.
